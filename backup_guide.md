# Настройка бэкапов базы данных (PostgreSQL + Docker)

Поскольку ваша база данных работает в Docker, самый надежный способ сделать бэкап — использовать утилиту `pg_dump`. Это создаст SQL-файл со всеми данными, который легко восстановить.

## 1. Ручной бэкап (Прямо сейчас)
Выполните эту команду на VPS, чтобы создать мгновенный снимок всей базы:

```bash
docker exec -t <имя_контейнера_db> pg_dump -U taurus test_db > ~/backup_$(date +%Y-%m-%d_%H-%M-%S).sql
```
*Замените `<имя_контейнера_db>` на имя вашего контейнера с базой (обычно это `telegram_bot-db-1` или просто `db`).*

## 2. Автоматические бэкапы (Cron)
Чтобы не делать это руками, настроим скрипт, который будет запускаться каждую ночь.

### Шаг А: Создание скрипта
Создайте файл `backup.sh` на VPS:
```bash
#!/bin/bash
# Настройки
CONTAINER_NAME="telegram_bot-db-1"  # Имя контейнера базы
DB_USER="taurus"
DB_NAME="test_db"
BACKUP_DIR="/home/taurus/backups"   # Папка для бэкапов
RETENTION_DAYS=7                     # Сколько дней хранить старые бэкапы

# Создаем папку, если её нет
mkdir -p $BACKUP_DIR

# Имя файла с датой
FILENAME="$BACKUP_DIR/db_backup_$(date +%Y-%m-%d_%H%M%S).sql"

# Выполняем дамп
docker exec -t $CONTAINER_NAME pg_dump -U $DB_USER $DB_NAME > $FILENAME

# Сжимаем бэкап (опционально, экономит место)
gzip $FILENAME

# Удаляем старые бэкапы (старше 7 дней)
find $BACKUP_DIR -type f -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "Backup created: $FILENAME.gz"
```

### Шаг Б: Разрешение на запуск
```bash
chmod +x backup.sh
```

### Шаг В: Настройка расписания
Откройте планировщик:
```bash
crontab -e
```
Добавьте в конец строку для запуска каждый день в 03:00 ночи:
```bash
0 3 * * * /home/taurus/backup.sh >> /home/taurus/backup.log 2>&1
```

## 3. Как восстановить данные из бэкапа?
Если что-то случится, восстановить базу можно одной командой:
```bash
cat <имя_файла>.sql | docker exec -i <имя_контейнера_db> psql -U taurus -d test_db
```
> [!CAUTION]
> Восстановление перезапишет данные в текущей базе. Всегда лучше сначала протестировать восстановление на тестовой БД.

## 4. Совет по безопасности
> [!TIP]
> Старайтесь копировать эти SQL-файлы с VPS на другой сервер или в облако. Если "умрет" сам диск VPS, локальные бэкапы не помогут. Можно использовать инструменты вроде `rclone` для отправки в Google Drive или Telegram S3.
