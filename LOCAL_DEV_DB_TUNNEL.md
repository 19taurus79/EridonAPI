# Інструкція: Робота з БД на VPS при локальній розробці (через SSH-туннель)

Оскільки база даних на вашому VPS з міркувань безпеки закрита для зовнішніх підключень, для локальної розробки (щоб ваш локальний код міг працювати з реальними даними на сервері) необхідно використовувати **SSH-туннель**.

## 1. Як це працює?
SSH-туннель створює безпечний міст між вашим комп'ютером і VPS. Ви кажете своєму комп'ютеру: *"Слухай локальний порт 5435 і перенаправляй весь трафік з нього через SSH на VPS, а там передавай його на базу даних"*.

В результаті ваш локальний бекенд думає, що база даних працює у вас на комп'ютері, хоча насправді вона на VPS.

## 2. Налаштування `.env` файлу (Один раз)
У вашому локальному файлі `.env` (в папці `EridonAPI`) переконайтеся, що налаштування вказують на локальний хост та порт туннелю:
```env
POSTGRES_HOST='127.0.0.1'
POSTGRES_PORT='5435'
```
*(Пароль і назва БД залишаються тими ж, що й на сервері).*

## 3. Як запускати міст (Туннель)
Кожного разу, коли ви сідаєте за локальну розробку, **перед запуском бекенда** вам потрібно відкрити туннель:

1. Відкрийте окреме вікно термінала (PowerShell) у вашій IDE або системі.
2. Введіть команду (використовуючи ваш профіль з `~/.ssh/config`):
   ```bash
   ssh -N -L 5435:127.0.0.1:5435 vps-195-port22222
   ```
3. Після натискання Enter термінал просто "зависне". **Це нормально!** Міст працює.
4. **Не закривайте це вікно і не натискайте `Ctrl+C`**, поки не закінчите розробку.

## 4. Запуск бекенда та міграцій
Після того як туннель відкрито (у фоновому терміналі), ви можете відкрити **інший** термінал і спокійно виконувати будь-які команди для БД:

- Запуск міграцій:
  ```bash
  .venv\Scripts\piccolo.exe migrations new new_agri_bot_backend --auto
  .venv\Scripts\piccolo.exe migrations forwards new_agri_bot_backend
  ```
- Запуск локального сервера FastAPI:
  ```bash
  .venv\Scripts\python.exe run.py
  ```

## 5. Що робити, якщо помилка `[WinError 1225] The remote computer refused the network connection`?
Ця помилка означає, що ваш бекенд намагається постукати у двері 5435 на вашому комп'ютері, але ніхто не відкриває. 
**Причина:** SSH-туннель не запущено або він розірвався.
**Рішення:** Переконайтеся, що вікно з командою `ssh -N -L ...` відкрито і туннель активний.
